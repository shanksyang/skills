# Notion 数据库 Schema

周报生成器使用的 Notion 数据库属性定义。

## 属性列表

| 属性名 | 类型 | 说明 | 周报用途 |
|--------|------|------|----------|
| `title` | title | 页面标题 | 作为周报条目标题 |
| `Date` | date | 日期/时间 | 按周筛选笔记的依据 |
| `领域` | select | 所属领域 | 过滤条件：仅查询 `🏢工作` |
| `分类` | multi_select | 内容分类 | 辅助分类（当前版本基于标题关键词） |
| `标签` | multi_select | 标签 | 辅助信息 |

## 查询逻辑

```python
# 周报生成器的 Notion 查询
response = notion.databases.query(
    database_id=database_id,
    filter={
        "property": "领域",
        "select": {
            "equals": "🏢工作"
        }
    }
)
```

查询后在本地按 `Date` 属性筛选日期范围内的笔记。

## 周报分类规则

当前基于标题关键词分类：

| 关键词 | 分类 |
|--------|------|
| 生活、聚餐 | 生活记录 |
| 管理、例会、周会 | 业务活动 |
| 其他 | 客户对接 |

## 周报输出格式

```markdown
**作者周报**

**2026年03月第9周**（0302-0308）

**1、本周总结**

本周共N项工作活动。

**1.1、客户对接**
● 一（0302）：标题
AI 总结摘要

**1.2、业务活动**
● 三（0304）：标题
AI 总结摘要
```
